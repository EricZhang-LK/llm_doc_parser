# ADR-008: 采用 FastAPI 作为 Web 框架并全面容器化

## 状态
已采纳 (2026-06-08)

## 决策
1. 使用 FastAPI 构建 RESTful API，利用其原生异步特性对接 RAG 异步管道。
2. 使用 Pydantic 进行严格的请求/响应数据校验。
3. 采用 Docker + docker-compose 进行全栈（API + Qdrant）编排与交付。

## 理由
- FastAPI 的性能和异步支持与我们的全异步 RAG 架构完美契合。
- 自动生成的 OpenAPI 文档极大降低了前后端联调成本。
- 容器化彻底解决了环境一致性问题，为 CI/CD 和 K8s 部署铺平道路。

## 后果
- (+) 提供了标准化的 HTTP 接口，支持任何语言的前端或客户端调用。
- (+) 实现了真正的“一次构建，到处运行”。
- (-) 增加了 Docker 镜像构建和维护的复杂度。

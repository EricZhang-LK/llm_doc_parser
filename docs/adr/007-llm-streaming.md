# ADR-007: 采用抽象基类封装 LLM 并支持异步流式输出

## 状态
已采纳 (2026-06-05)

## 决策
1. 定义 `BaseLLM` 抽象类，统一 `chat` 和 `chat_stream` 接口。
2. 使用 `AsyncGenerator` 实现非阻塞的流式文本生成。
3. 在 `RAGPipeline` 中同步暴露流式接口，确保端到端的流式体验。

## 理由
- 抽象层消除了业务代码对特定 LLM 厂商 SDK 的强依赖，方便未来扩展多模型路由。
- 异步流式输出（AsyncGenerator）是 Python 高并发网络服务的标准范式，能最大化服务器吞吐量。
- 显著降低了 RAG 应用在长文本生成时的首字延迟（TTFT），提升用户体验。

## 后果
- (+) 系统具备了生产级 AI 应用必备的流式响应能力。
- (+) 代码具备极强的扩展性，接入 Claude 或本地大模型仅需新增一个适配器类。
- (-) 增加了异步生成器的复杂度，调用方需要使用 `async for` 进行消费。
